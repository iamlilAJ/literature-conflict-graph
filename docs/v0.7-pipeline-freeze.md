# aigraph v0.7 Pipeline Freeze Record

Frozen on 2026-05-08. Subsequent commits must NOT change the listed
modules until a thaw condition (§4) is met.

The intent: lock the **idea-generation** pipeline (claim extraction →
typed graph → anomaly detection → hypothesis generation → influence
scoring → MMR selection) so that controlled-validation experiments
produce ρ numbers comparable across runs and across cohorts.

The data-source layer (`corpus.py`, future `intern_atlas_loader.py`)
and tooling (`scripts/`, `docs/`, `tests/`) remain freely editable —
they don't affect the predictor's identity, only what gets fed in.

## 1. Frozen modules

The following are sealed for v0.7 controlled-validation:

| Module | Stage | Why frozen |
|---|---|---|
| `src/aigraph/extract.py` | [3] claim extraction | Atomic-claim granularity contract; downstream graph topology assumes 1 claim = 1 method-on-task statement |
| `src/aigraph/llm_extract.py` | [3] claim extraction | LLM prompt + output schema; changing it changes the per-paper claim count distribution |
| `src/aigraph/graph.py` | [4] typed graph | Node-type taxonomy (Paper/Claim/Method/Task) and edge-type taxonomy (cites/extends/uses/about); both feed anomaly detectors |
| `src/aigraph/anomalies.py` | [5] anomaly detection | The 8 anomaly types (§2) ARE the differentiator; their definitions are the contract |
| `src/aigraph/hypotheses.py` | [6] template fallback | Template strings + content-free moderator template; affects what the deterministic fallback emits |
| `src/aigraph/llm_hypotheses.py` | [6] LLM hypothesis generation | 8 type-specific prompts per anomaly type; freezing the prompt freezes hypothesis style |
| `src/aigraph/creator.py` | [6] multi-grain creator | fine/coarse/synthesize three-stage logic + open-question integration |
| `src/aigraph/influence.py` | [7] Phase 1 4-dim scoring | Dimension definitions + `WEIGHTS_PHASE1` (§3); changing weights invalidates ρ comparison across runs |

`src/aigraph/models.py` is **not** frozen — adding optional fields to
data-layer schemas is a passive change that doesn't affect predictor
behavior. New fields simply get default values for existing rows.

## 2. The 8 anomaly types (snapshot)

Frozen contract from `src/aigraph/models.py:AnomalyType` Literal +
`src/aigraph/anomalies.py` detectors:

1. **`benchmark_inconsistency`** — within a (method, task) cluster,
   both positive and negative result claims exist. Asks "When does
   {method} help vs hurt {task}?". Detector: `_detect_benchmark_inconsistency`.
2. **`impact_conflict`** — special case of the above: the disagreement
   exists between high-impact papers (positive impact ≥ 2.0 AND
   negative impact ≥ 2.0). Asks "Why do high-impact papers disagree
   about {method} on {task}?". Same detector path, different label.
3. **`setting_mismatch`** — same (method, task) but different settings
   (top-k, retrieval depth, batch size, etc.) drive divergent results.
   Detector: `_detect_setting_mismatch`.
4. **`metric_mismatch`** — same (method, task) but different metrics
   (EM vs F1, accuracy vs ROUGE) yield incompatible rankings.
   Detector: `_detect_metric_mismatch`.
5. **`evidence_gap`** — a (method, task) cluster has fewer claims
   than its anchoring entities suggest; missing evidence around the
   pair. Detector: `_detect_evidence_gap`.
6. **`community_disconnect`** — two methods or tasks have shared
   underlying mechanisms but no co-citation or co-mention path.
   Detector: `_detect_community_disconnect`.
7. **`bridge_opportunity`** — A and B are heavily co-cited but no
   single paper directly extends one with the other. Detector:
   `_detect_bridge_opportunity`.
8. **`replication_conflict`** — repeated independent attempts to
   measure the same effect produce divergent magnitudes. Detector:
   `_detect_replication_conflict`.

The 8-type taxonomy is the load-bearing differentiator from
`docs/intern-atlas-pivot.md` §4.1: Atlas's `paper_evolution_edges`
expose extends/improves/replaces/adapts/uses_component/compares/
background — all causal/derivational, none capturing
disagreement-about-the-same-effect (see also
`docs/intern-atlas-schema-compat.md` §2.1).

## 3. Influence Phase 1 weights (snapshot)

From `src/aigraph/influence.py:WEIGHTS_PHASE1`:

| Dimension | Weight | Code key |
|---|---:|---|
| I_2 Community Reach | 0.25 | `community_reach` |
| I_4 Novelty vs External | 0.30 | `novelty` |

## 4. Thaw conditions

Only these justify modifying the frozen modules:

1. **Predictor fundamentally broken**: ρ_total < 0.10 on the primary
   cohort (NeurIPS/ICML/ICLR 2018-2020, 1790 papers — see
   `docs/controlled-validation-design.md §3.1` post-pivot amendment).
   This bar is well below the v3 _community-fixture result (ρ=0.603)
   and far below the design's 0.30 success bar; if it triggers, the
   scoring formula or weights are meaningfully wrong and a thaw is
   warranted.
2. **Anomaly type produces 0 detections** on a 1000+ paper real
   cohort. Each detector should fire at least once on that scale.
   Zero detections means either the detector has a real bug or its
   threshold is too restrictive.
3. **Reviewer-mandated ablation** in paper writeup. If a reviewer asks
   for a specific ablation that requires modifying frozen code, the
   ablation lives on a branch + the v0.7-frozen tag stays the
   reference predictor.

NOT thaw conditions:
- "I have a new idea I want to try"
- "Atlas added more data"
- "I saw a better approach in another paper"
- "The selected hypothesis on topic X looks weak"

The freeze deliberately prevents ad-hoc tuning during the validation
window. Tuning rounds happen AFTER one full validation pass produces
numbers we agree on.

## 7. Thaw record

Each authorized thaw of a frozen module is logged here, with the §4
condition that fired, the diff, the empirical before/after, and the
reasoning. Reviewers can audit each thaw by reading this section
chronologically.

### Thaw #1 — 2026-05-10 — `bridge_opportunity` detector threshold

**Fired condition (§4):** condition 2 — anomaly type produces 0
detections on a 1000+ paper real cohort.

**Empirical trigger.** On the 540-paper full-text run
(`artifacts/runs/arxiv-reasoning-v0.7-540p`), `bridge_opportunity`
produced **0 anomalies** despite 198 (method, task) clusters and
~19,500 candidate cluster pairs. Diagnostic instrumentation showed
99.9% of pairs (19,491 / 19,503) were dropped at the
`meaningful_overlap < _BRIDGE_MIN_MEANINGFUL_TOKENS` filter (constant
was 3).

**Root cause.** `(method, task)` keys in this corpus tokenize to
4–5 meaningful tokens after stripping `_GENERIC_CONCEPT_TOKENS`.
Requiring 3-overlap effectively forces near-identical pairs (sharing
3 of ~4 tokens), which is too tight for a bridge-detection intent
that should fire when two clusters share **either** the method OR
the task domain.

**Diff.**

```diff
-_BRIDGE_MIN_MEANINGFUL_TOKENS = 3
+_BRIDGE_MIN_MEANINGFUL_TOKENS = 2
```

That is the entire change. The other two bridge constants
(`_BRIDGE_MIN_CLUSTER_PAPERS = 5`,
`_BRIDGE_REFERENCED_WORKS_COVERAGE = 0.5`) and the citation-gate
logic are unchanged. Inline comment in `anomalies.py` records the
v0.7-frozen §7 reference.

**Empirical after.** On the same 540-paper graph + claims
(no other input changed), re-running `aigraph detect-anomalies`:

```
                          before    after    delta
bridge_opportunity             0       40      +40
... (other 7 types unchanged)
TOTAL anomalies              456      496      +40
```

Other 7 detectors produced byte-identical output. Sampled three
bridge anomalies — all are reasonable cross-method-same-task pairs
(e.g. `agent` ↔ `multimodal` on `multi-step-reasoning`,
`evaluation-method` ↔ `agent` on `multi-step-reasoning`).

**Why this is a legitimate thaw, not a tuning preference.** §4
condition 2 is the explicit purpose of this knob: a detector that
is silent at scale on a properly-formed cohort is, by the freeze
contract, a bug or a too-tight threshold. The pre-thaw constant
behaves correctly only on much shorter (method, task) keys (e.g.
single-word keys with rich vocabularies). Our 2026 reasoning corpus
has compound keys that shrink the meaningful-token budget. 2-overlap
is the smallest relaxation that makes the detector fire.

**Validation pass.** Full pytest suite: 250 passed, 1 skipped (no
regression). `tests/test_anomalies.py` 19/19 pass — no test pinned
the old threshold.

**Tagged downstream.** `artifacts/runs/arxiv-reasoning-v0.7-540p/`
was produced against pre-thaw `4dad598`. Future runs against
`v0.7-frozen` (post-thaw tip) will pick up this change. Backtest
ρ values quoted in
`/tmp/influence_backtest_comparison.md` are pre-thaw and should be
re-computed if/when bridge_opportunity hypotheses become part of
the validation set.
